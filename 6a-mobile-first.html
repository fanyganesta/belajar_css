Sistem Adaptive (Responsive Design)

Sistem adaptive (responsive) adalah cara membuat website mampu menyesuaikan diri dengan:

  • ukuran layar (HP, tablet, laptop, monitor besar)
  • orientasi (portrait / landscape)
  • kemampuan perangkat
  • kebiasaan pengguna
Tujuan utamanya:

Konten tetap nyaman dibaca, digunakan, dan diakses di perangkat apa pun.

Responsive bukan sekadar mengecilkan tampilan,
tetapi menyusun ulang ruang dan prioritas informasi.

Masalah yang diselesaikan Responsive

Tanpa responsive:

  • teks terlalu kecil di HP
  • tombol susah diklik
  • sidebar memakan layar
  • layout pecah
Dengan responsive:
  • satu kode ‚Üí banyak perangkat
  • UX konsisten
  • lebih SEO friendly
  • lebih mudah dirawat

Mobile First

Apa itu Mobile First?

Mobile First adalah pendekatan desain di mana:

Kita mendesain untuk layar kecil terlebih dahulu, baru menambah fitur untuk layar besar.

Bukan:

Melainkan:

‚ÄúBikin versi HP yang paling penting dulu, lalu diperluas‚Äù

Kenapa Mobile First itu penting?

Fakta dunia nyata:

70% trafik web berasal dari mobile

HP memiliki:

  • layar kecil
  • koneksi lebih lambat
  • CPU lebih lemah
  • sentuhan, bukan mouse
Jadi kita harus memprioritaskan:
  • keterbacaan
  • kesederhanaan
  • kecepatan

Mental model Mobile First

Bayangkan kamu sedang membuat aplikasi di HP:

  • ruang sangat terbatas
  • hanya konten paling penting yang boleh muncul
Itulah inti Mobile First:

memilih apa yang benar-benar penting

Mobile First dalam CSS

Default CSS kita anggap untuk mobile.

Lalu kita pakai media query untuk menambah aturan pada layar besar. Contoh:

    /* Mobile (default) */ 

        .card { 

        width: 100%; 

        font-size: 14px; 

    } 
 


    /* Tablet ke atas */ 

    @media (min-width: 768px) { 

        .card {  

            width: 50%; 

            font-size: 16px; 

        } 

    } 
 


    /* Desktop besar */ 

        @media (min-width: 1200px) { 

        .card { 

            width: 33.33%; 

        } 

    }

Kenapa pakai min-width, bukan max-width?

Mobile First menggunakan:

    @media (min-width: ...)

Artinya:

aturan berlaku mulai dari ukuran ini ke atas

Ini membuat CSS:

  • lebih ringan
  • lebih mudah dibaca
  • tidak perlu ‚Äúmenimpa‚Äù aturan mobile

Perbandingan pendekatan

Desktop First Mobile First
Mulai dari besar Mulai dari kecil
Menghapus fitur Menambah fitur
CSS berat CSS ringan
Sulit dirawat Mudah dikembangkan

Contoh nyata

Tanpa Mobile First:

  • bikin 3 kolom
  • lalu di HP dipecah jadi 1 kolom
Dengan Mobile First:
  • bikin 1 kolom
  • lalu di desktop ditingkatkan jadi 3 kolom
Secara psikologis, ini jauh lebih benar.

Intinya

Mobile First adalah:
  • Strategi berpikir sebelum menulis CSS
  • Bukan sekadar teknik, tapi cara menentukan prioritas desain.
