2. Inheritance Control (Pengendalian Pewarisan CSS)

Apa itu Inheritance?

Inheritance (pewarisan) adalah mekanisme di mana:

beberapa properti CSS diturunkan dari parent ke child secara otomatis

Contoh:

    body { 

        color: #333; 

        font-family: Arial, sans-serif; 

    }

Ini teks

Ini span

✔ color dan font-family diwariskan ke semua child

Properti yang Secara Default Diwariskan

Umumnya properti teks:

  • color
  • font-family
  • font-size
  • font-weight
  • line-height
  • text-align
  • visibility
❌ Properti layout tidak diwariskan:
  • margin
  • padding
  • border
  • width
  • height
  • display
  • position

Masalah Umum Akibat Inheritance

1️⃣ Warna atau Font Tidak Sengaja Menyebar

    .sidebar { 

        color: white; 

    } 
 


    <div class="sidebar"> 

    <button>Click</button> 

    </div>

✔ Button ikut putih, padahal tidak diinginkan

2️⃣ Komponen Jadi Tidak Isolatif

Komponen:

    .card { 

        font-size: 18px; 

    }

Jika .card dipindah ke tempat lain:

  • tampilannya bisa berubah
  • tidak konsisten

Mengontrol Inheritance dengan Sengaja

CSS menyediakan nilai khusus:

1️⃣ inherit

Memaksa properti mengikuti parent

    button { 

        color: inherit; 

    }

✔ Button mengikuti warna teks parent

2️⃣ initial

Mengembalikan ke nilai default CSS

    button { 

        color: initial; 

    }

✔ Tidak peduli parent

3️⃣ unset (Paling Fleksibel)

Jika properti diwariskan → inherit

Jika tidak → initial

    button { 

        font-size: unset; 

    }

✔ Sangat cocok untuk reset komponen

4️⃣ revert (Advanced)

Mengembalikan ke:

  • user-agent stylesheet
  • atau stylesheet sebelumnya

    button { 

        all: revert; 

    }

⚠ Biasanya dipakai untuk reset berat

Properti Khusus: all

    .card { 

        all: unset; 

    }

⚠ Reset total semua properti

Biasanya digunakan untuk:

  • Web Component
  • Isolasi ekstrem

Strategi CSS Engineer dalam Inheritance

1️⃣ Base Style di Root

    :root { 

        font-family: system-ui, sans-serif; 

        color: #222; 

    }

✔ Semua komponen konsisten

2️⃣ Komponen Tidak Bergantung Parent

❌ Salah:

    .card { 

        color: inherit; 

    }

✔ Benar:

    .card { 

        color: var(--text-primary); 

    }

3️⃣ Reset Selektif, Bukan Global

❌ Salah:

    * { 

        color: black; 

    }

✔ Benar:

    button { 

        font: inherit; 

    }

Kasus Nyata: Button Konsisten

    button { 

        font-family: inherit; 

        color: inherit; 

        background: none; 

        border: none; 

    }

✔ Button mengikuti konteks tanpa bug

Inheritance vs Cascade

Inheritance Cascade
Pewarisan dari parent Konflik antar selector
Otomatis Berdasarkan aturan
Hanya properti tertentu Semua properti

CSS Engineer harus paham keduanya.

Checklist Inheritance Control

❓ Properti ini diwariskan atau tidak?
❓ Komponen bergantung parent?
❓ Perlu inherit, unset, atau initial?
❓ Apakah komponen reusable?

Ringkasan Inheritance Control

  • Inheritance bisa membantu atau merusak
  • Gunakan dengan sengaja
  • Komponen ideal = tidak bergantung konteks
  • inherit, unset, initial adalah senjata penting
  • CSS Engineer mengontrol pewarisan, bukan membiarkannya
