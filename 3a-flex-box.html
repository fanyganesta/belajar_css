CSS Satu Arah ‚Äì Flexbox

1. Mengapa Flexbox Ada?

Sebelum Flexbox, layout CSS dibuat dengan:

  • float
  • inline-block
  • position
  • table

Semua itu bukan dibuat untuk layout, tapi dipaksa untuk layout.
Akibatnya:

  • susah sejajar
  • vertikal center ribet
  • responsive berantakan
  • perlu banyak hack
Flexbox diciptakan untuk satu tujuan:

Mengatur tata letak elemen dalam satu arah secara fleksibel dan otomatis.

2. Apa maksud ‚ÄúCSS satu arah‚Äù?

Flexbox bekerja dalam satu dimensi:

  • horizontal atau
  • vertikal
  • Tidak bisa dua arah sekaligus.
üìå Itulah sebabnya disebut:

Layout satu arah (one-dimensional layout system)

Kalau mau dua arah ‚Üí CSS Grid.

3. Konsep Dasar Flexbox

Flexbox selalu terdiri dari:

  • Flex Container
  • Flex Items

    .container { 

        display: flex; 

    }

Saat sebuah elemen diberi display: flex, maka:

  • dia menjadi flex container
  • semua anak langsungnya menjadi flex items

HTML: 
    <div class="container">
    <div>A</div>
    <div>B</div>
    <div>C</div>
    </div>
CSS: 
    .container { 

        display: flex; 

    }

‚û° A, B, C otomatis sejajar dalam satu baris

  • arah

  • dan menyusunnya secara otomatis

5. Filosofi Flexbox

Flexbox bukan tentang:

‚Äúdi mana posisi elemen ini?‚Äù

tetapi:

‚Äúbagaimana ruang dibagi ke elemen-elemen ini?‚Äù

Ini membuat:

  • center otomatis
  • distribusi merata
  • responsif tanpa media query
  • sejajar tanpa kalkulasi manual

6. Perbedaan dengan width & height biasa

Tanpa Flexbox:

    .box { 

        width: 33%; 

    }

Jika satu box lebih besar ‚Üí layout rusak.

Dengan Flexbox:

    .container { 

        display: flex; 

    }

Browser:

  • membagi ruang
  • menyesuaikan
  • merapikan secara otomatis

7. Kapan Flexbox dipakai?

Flexbox cocok untuk:

  • navbar
  • sidebar + content
  • card list
  • form layout
  • button group
  • dashboard
  • header/footer
Tidak cocok untuk:
    layout dua arah kompleks (pakai Grid)

Saat memakai Flexbox, kita tidak berpikir:

‚ÄúBagaimana elemen-elemen ini dibagi di dalam container?‚Äù

Itu sebabnya properti Flexbox selalu berbicara tentang:

  • alignment
  • distribusi
  • arah
  • proporsi
  • bukan koordinat

9. Gambaran besar

Flexbox itu:

  • sistem tata letak berbasis aliran dan ruang
  • Bukan posisi mutlak.

10. Ringkasan

Flexbox adalah:

  • sistem layout satu arah
  • berbasis container
  • otomatis mengatur anak-anaknya
  • fokus pada pembagian ruang
  • dibuat khusus untuk UI modern
