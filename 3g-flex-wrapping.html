Flex-Wrapping (flex-wrap)

1. Masalah tanpa flex-wrap

Secara default Flexbox adalah satu baris saja:

    .container { 

        display: flex; 

    }

Artinya:

Semua item dipaksa berada dalam satu baris, walaupun kepadatan tinggi

Akibatnya:

  • Item mengecil (flex-shrink)
  • Atau keluar container
  • Atau teks rusak

2. Fungsi flex-wrap

flex-wrap menentukan:

Bolehkah item turun ke baris berikutnya?

    .container { 

        display: flex; 

        flex-wrap: wrap; 

    }

Sekarang:

Jika tidak muat → item pindah ke baris baru

3. Nilai flex-wrap

Nilai Fungsi
nowrap Default, satu baris
wrap Boleh turun ke baris baru
wrap-reverse Sama seperti wrap, tapi arah cross-axis dibalik

4. Contoh nyata

Galeri kartu

    .container { 

        display: flex; 

        flex-wrap: wrap; 

        gap: 16px; 

    } 
 


    .card { 

        flex: 0 0 200px; 

    }

➡ Kartu akan tersusun grid otomatis
➡ Kalau layar kecil → turun ke baris berikutnya

5. Flex-wrap vs Grid

Flexbox Grid
Satu arah Dua arah
Cocok navbar & card list Cocok layout halaman
Wrap manual Grid otomatis

Flexbox + wrap ≈ grid ringan

6. Hubungan dengan flex-basis

Wrap bekerja berdasarkan flex-basis

    .card { 

        flex: 0 0 250px; 

    }

Artinya:

Satu kartu minimal 250px sebelum pindah baris

7. Kesalahan umum

❌ Mengira flex otomatis bikin grid
❌ Lupa flex-wrap: wrap
❌ Tidak set flex-basis

8. Pola profesional

Responsive card layout

    .container { 

        display: flex; 

        flex-wrap: wrap; 

    } 
 


    .card { 

        flex: 1 1 250px; 

    }

Artinya:

  • Minimal 250px
  • Bisa membesar
  • Bisa turun baris
Ini adalah pola paling sering dipakai di web modern.
