3. Naming Convention (BEM & Utility)

Naming convention adalah aturan penamaan class CSS agar:

  • mudah dibaca
  • tidak bentrok
  • scalable
  • aman untuk tim besar
Tanpa naming convention, CSS cepat berubah menjadi chaos.

Masalah CSS tanpa aturan penamaan

    .box { } 

    .box2 { } 

    .box-red { } 

    .box-big { } 

    .bigBox { }

Masalahnya:

  • tidak jelas fungsinya
  • sulit ditebak hubungannya
  • rawan konflik
  • sulit dirawat

Solusi: Naming Convention

Dalam Design System modern, dua pendekatan paling populer:

  • BEM (Block Element Modifier)
  • Utility-based naming
  • Biasanya dipakai bersamaan.

A. BEM (Block – Element – Modifier)

BEM adalah metode penamaan berbasis komponen.

Struktur BEM

    .block 

    .block__element 

    .block--modifier

1. Block

Komponen utama (mandiri)

    .card { } 

    .button { } 

    .navbar { }

2. Element

Bagian dari block (tidak berdiri sendiri)

    .card__title { } 

    .card__content { } 

    .card__footer { }

HTML:

    <div class="card"> 

    <h2 class="card__title">Title</h2> 

    <p class="card__content">Text</p> 

    </div>

3. Modifier

Variasi dari block atau element

    .card--highlight { } 

    .button--primary { } 

    .button--disabled { }

HTML:

    <button class="button button--primary">Save</button>

Aturan penting BEM

✔ Jangan lompat struktur
❌ .card__title__text
✔ .card__title

✔ Modifier tidak berdiri sendiri
❌ .button--primary
✔ .button button--primary

Kapan BEM sangat cocok?

  • Component-based CSS
  • UI besar
  • Tim lebih dari 1 orang
  • Framework modern

B. Utility Naming

Utility class adalah class satu fungsi satu tugas.

   
    .mt-4 { margin-top: 1rem; } 

    .text-center { text-align: center; } 

    .d-flex { display: flex; }

HTML:

    <div class="card mt-4 text-center">

Kelebihan Utility

  • cepat
  • fleksibel
  • minim CSS baru
  • cocok untuk layout & spacing

Kekurangan jika murni utility

    <div class="p-4 bg-white rounded shadow w-full">

HTML jadi panjang

makna desain tidak terlihat

Best Practice Modern

Komponen untuk struktur & makna
Utility untuk layout & spacing

Contoh:

    <div class="card mt-6"> 

    <h2 class="card__title text-center">Title</h2> 

    </div>

Perbandingan Singkat

BEM Utility
Berbasis komponen Berbasis fungsi
Konsisten Fleksibel
CSS lebih panjang HTML lebih panjang
Mudah dipahami tim Cepat prototyping

Ringkasan

  • Naming convention = pondasi CSS besar
  • BEM → struktur & komponen
  • Utility → layout & spacing
  • Digabung = sistem paling kuat
