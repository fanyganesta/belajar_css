Flex-grow, Flex-shrink, dan Flex-basis (flex)

1. Apa itu flex?

Di Flexbox, ukuran item tidak ditentukan hanya oleh width, tapi oleh tiga hal:

Properti Funsgi
flex-grow Seberapa besar item boleh membesar
flex-shrink Seberapa besar item boleh mengecil
flex-basis Ukuran dasar awal item

Ketiganya digabung menjadi:

    flex: grow shrink basis;

Contoh:

    .item { 

        flex: 1 1 200px; 

    }

2. flex-basis — ukuran awal

flex-basis adalah lebar awal item sebelum ruang dibagi.

    .item { 

        flex-basis: 200px; 

    }

Artinya:

“Anggap item ini lebarnya 200px dulu”

Ini lebih kuat daripada width di Flexbox.

3. flex-grow — membagi ruang sisa

Jika container punya ruang kosong, flex-grow menentukan:

Siapa dapat ruang ekstra dan berapa banyak

Contoh:

    .item1 { flex-grow: 1; } 

    .item2 { flex-grow: 2; }

Jika ada 300px ruang kosong:

  • item1 dapat 100px
  • item2 dapat 200px

➡ Dibagi berdasarkan rasio

4. flex-shrink — siapa yang menyusut

Jika container kekurangan ruang, item akan dipaksa mengecil.

    .item1 { flex-shrink: 1; } 
 
    .item2 { flex-shrink: 0; }

➡ item1 mengecil
➡ item2 tetap

5. Kenapa grow dan shrink tidak bertentangan?

Mereka bekerja pada kondisi berbeda:

  • Kondisi Yang aktif
  • Ada ruang kosong = flex-grow
  • Kurang ruang = flex-shrink
Jadi mereka tidak saling melawan, tapi bergantian.

6. flex: 1 artinya apa?

 
    .item { 

        flex: 1; 

    }

Itu artinya:

    flex: 1 1 0%;

Maknanya:

  • semua item sama besar
  • semua boleh membesar
  • semua boleh mengecil
  • basis awal = 0

➡ Paling sering dipakai untuk layout kolom.

7. Contoh nyata

Layout 3 kolom 
    .col { 

        flex: 1; 

    }

➡ Semua kolom sama lebar.

Sidebar + konten 
    .sidebar { flex: 0 0 250px; } 

    .content { flex: 1; }

➡ sidebar tetap
➡ konten mengisi sisa layar

8. Kesalahan umum

❌ Pakai width di Flexbox
❌ Tidak mengatur shrink → item hancur
❌ Salah paham flex:1

9. Inti penting

Flexbox tidak bekerja pakai ukuran statis — tapi sistem pembagian ruang.
