Hide / Show Element (Konten Adaptif)

Responsive bukan cuma soal mengatur ukuran,
tapi juga memilih apa yang perlu ditampilkan.

Kadang:

konten desktop tidak cocok ditampilkan di mobile.

Kenapa perlu hide / show?

Di HP:

  • layar sempit
  • interaksi sentuh
  • perhatian pengguna pendek
Maka kita:
  • sembunyikan sidebar
  • ganti menu jadi ikon
  • ringkas tabel
  • kurangi dekorasi

Teknik paling dasar

    .mobile-only { 

        display: block; 

    } 
 


    .desktop-only { 

        display: none; 

    } 


    @media (min-width: 768px) { 

        .mobile-only { 

            display: none; 

    } 
 


    .desktop-only { 

            display: block; 

        } 

    }

Hide dengan visibility

    .hidden { 

        visibility: hidden; 

    }

Elemen:

  • tidak terlihat
  • masih memakan ruang
Berbeda dengan:

    display: none;

Yang benar-benar menghilangkan.

Hide dengan opacity

    .fade { 

        opacity: 0; 

        pointer-events: none; 

    }

Dipakai untuk animasi atau transisi.

Responsive menu (contoh nyata)

    .menu { 

        display: none; 

    } 
 


    @media (min-width: 768px) { 

        .menu { 

            display: flex; 

        } 

    } 
 


    .hamburger { 

        display: block; 

    } 
 


    @media (min-width: 768px) { 

        .hamburger { 

            display: none; 

        } 

    }

HP → tombol hamburger
Desktop → menu penuh

Best practice

Jangan menyembunyikan:

  • konten utama
  • informasi penting
  • form wajib
Yang disembunyikan:
  • dekorasi/li>
  • menu sekunder
  • filter lanjutan

Aksesibilitas

Jika elemen disembunyikan:

  • screen reader juga tidak membacanya jika display:none
  • gunakan dengan sadar
