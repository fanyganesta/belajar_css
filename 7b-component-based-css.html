2. Component-Based CSS

Component-based CSS adalah pendekatan di mana:

UI dibangun dari potongan-potongan kecil yang mandiri, bisa dipakai ulang, dan konsisten.

Seperti Lego:

  • Button
  • Card
  • Alert
  • Modal
  • Input

Bukan halaman → tapi komponen.

Masalah CSS tanpa komponen

Biasanya terlihat seperti ini:

    .home button { ... } 

    .profile button { ... } 

    .admin button { ... }

Artinya:

  • style tergantung halaman
  • sulit reuse
  • perubahan satu tombol bisa merusak halaman lain

Cara berpikir component-based

Bukan:

“tombol di halaman A”

Tapi:

“komponen Button”

Contoh: Button sebagai komponen

    .btn { 

        padding: var(--space-sm) var(--space-md); 

        border-radius: var(--radius); 

        font-size: 1rem; 

        cursor: pointer; 

        border: none; 

    }

Lalu variasi:

    .btn-primary { 

        background: var(--color-primary); 

        color: white; 

    } 
 


    .btn-secondary { 

        background: var(--color-secondary); 

    }

HTML:

    <button class="btn btn-primary">Save</button> 

    <button class="btn btn-secondary">Cancel</button>

Komponen tidak tahu di mana dia dipakai

Button tidak peduli:

  • di sidebar
  • di modal
  • di form
Itulah kekuatan sistem.

Komponen + CSS Variable

    .card { 

        background: var(--card-bg, white); 

        padding: var(--space-md); 

        border-radius: var(--radius); 

    }

Setiap komponen bisa:

  • pakai token global
  • override token lokal

Keuntungan besar

Dengan component-based CSS:

  • UI konsisten
  • bug lebih sedikit
  • scalable
  • cocok untuk tim
  • cocok untuk React, Vue, Laravel Blade, dll

Perbedaan dengan utility-only

Component-based:

“Saya pakai Card”

Utility:

“Saya susun Card dari 10 class”

Biasanya dipakai bersama dalam sistem modern.

Ringkas

Component-based CSS mengubah:

  • halaman → menjadi kumpulan komponen
  • Dan design system hanya bisa hidup jika UI dibuat dari komponen
