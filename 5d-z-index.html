Stacking Context (konsep kunci)

Ini bagian yang paling sering menyebabkan:

‚Äúz-index saya besar tapi tetap kalah üò≠‚Äù

Apa itu stacking context?

Stacking context adalah dunia z-index sendiri.

Jika elemen membuat stacking context baru,
anak-anaknya tidak bisa menembus keluar.

Elemen yang membuat stacking context baru:

  • position + z-index (selain auto)
  • opacity < 1
  • transform
  • filter
  • will-change
  • position: fixed
  • position: sticky (kadang)

Contoh masalah klasik

    .parent { 

        position: relative; 

        z-index: 1; 

    } 
 


    .child { 

        position: absolute; 

        z-index: 9999; 

    }

Jika ada elemen lain di luar .parent dengan:

    z-index: 2;

üëâ Elemen luar tetap di atas, walaupun child punya 9999

Kenapa?
Karena child terkunci di stacking context parent.

Ilustrasi mental

Bayangkan:

  • setiap stacking context = kotak kaca
  • z-index hanya berlaku di dalam kotak itu

    .box { 

        position: relative; 

        z-index: -1; 

    }
