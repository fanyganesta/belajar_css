A. Pengantar: Mengatur Ruang dengan CSS

Pada level intermediate, CSS tidak lagi sekadar:

  • memberi warna
  • mengatur font
  • menambahkan dekorasi
Tetapi mulai fokus pada:
  • bagaimana elemen mengambil ruang
  • bagaimana elemen berinteraksi satu sama lain
  • bagaimana layout tetap stabil di berbagai ukuran layar
üìå Tujuan utama:
Mengontrol ruang, ukuran, dan perilaku elemen secara presisi dan konsisten.

Mengapa ‚ÄúMengatur Ruang‚Äù itu penting?
Tanpa pengaturan ruang yang baik:
  • layout mudah rusak
  • elemen saling bertabrakan
  • desain sulit responsif
  • maintenance CSS menjadi rumit
CSS menyediakan beberapa mekanisme inti untuk mengatur ruang, antara lain:
  • Box Model
  • Display
  • Width & Height
  • Overflow

B. Box Model (Advanced)

1. Review Singkat Box Model

Setiap elemen HTML dipandang sebagai kotak (box).
Struktur Box Model:
Content ‚Üí Padding ‚Üí Border ‚Üí Margin

üìå Pada level dasar, kita hanya ‚Äúmenggunakan‚Äù box model.
üìå Pada level advanced, kita mengontrol perilakunya.

C. Masalah Klasik Box Model (Sebelum box-sizing)

Perhatikan kode berikut: 

    .box { 

        width: 300px; 

        padding: 20px; 

        border: 5px solid black; 

    }

‚ùì Pertanyaan penting:
Apakah lebar elemen ini 300px?
‚ùå Tidak
Lebar total elemen menjadi:

  • 300 (content)
  • + 40 (padding kiri + kanan)
  • + 10 (border kiri + kanan)
  • = 350px
üìå Inilah sumber masalah layout:
  • elemen melebar tanpa disadari
  • grid tidak sejajar
  • overflow horizontal muncul

D. box-sizing (Inti Box Model Advanced)

1. Apa itu box-sizing?

box-sizing menentukan:

  • bagian mana yang dihitung sebagai ukuran elemen
Properti ini mengontrol:
  • bagaimana width dan height bekerja
  • bagaimana padding & border mempengaruhi ukuran

2. Nilai box-sizing

a. content-box (Default)

    box-sizing: content-box;

Karakteristik:

  • width & height hanya untuk content
  • padding & border menambah ukuran
  • perilaku default browser
üìå Ini sering tidak diinginkan dalam layout modern.

b. border-box (Recommended)

    box-sizing: border-box;

Karakteristik:

  • width & height mencakup:
    • content
    • padding
    • border
    • ukuran elemen tetap konsisten
üìå Inilah pendekatan modern & profesional.

3. Perbandingan Visual

    .box { 

        width: 300px; 

        padding: 20px; 

        border: 5px solid black; 

    }
  • box-sizing Lebar total
  • content-box 350px
  • border-box 300px

4. Mengapa border-box Lebih Baik?

Dengan border-box:

  • layout lebih mudah diprediksi
  • padding tidak merusak grid
  • responsive design lebih stabil
  • developer tidak perlu menghitung manual
üìå Hampir semua framework modern (Bootstrap, Tailwind, dsb) menggunakan border-box.

5. Praktik Global box-sizing (Best Practice)

Best practice yang umum digunakan: 

    *, 

    *::before, 

    *::after { 

        box-sizing: border-box; 

    }

üìå Artinya:

  • semua elemen
  • termasuk pseudo-element
menggunakan box model yang konsisten.
