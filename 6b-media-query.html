Media Queries

Media Query adalah cara CSS berkata:

“Gunakan style ini hanya jika kondisi tertentu terpenuhi.”

Biasanya kondisinya adalah:

  • lebar layar
  • tinggi layar
  • orientasi
  • resolusi

Bentuk dasar

    @media (min-width: 768px) { 

        .box { 

            background: blue; 

        } 

    }

Artinya:

  • Jika lebar layar ≥ 768px, ubah .box menjadi biru
  • Jika lebih kecil → aturan ini diabaikan

Media Query = If di CSS

Secara mental:

    IF screen width >= 768px 

    apply this CSS

Breakpoint (titik perubahan)

Breakpoint adalah titik di mana layout berubah.

Umum dipakai:

Device Breakpoint
HP kecil 320–480px
HP besar 480–768px
Tablet 768–1024px
Laptop 1024–1200px
Desktop 1200px+

Biasanya kita pakai:

  • 768px
  • 1024px
  • 1200px

Mobile First + Media Query

Inilah kombinasi yang benar:

    /* Mobile default */ 

    .grid { 

        display: block; 

    } 
 


    /* Tablet ke atas */ 

    @media (min-width: 768px) { 

        .grid { 

            display: grid; 

            grid-template-columns: 1fr 1fr; 

        } 

    } 
 


    /* Desktop */ 

    @media (min-width: 1200px) { 

        .grid { 

            grid-template-columns: 1fr 1fr 1fr; 

        } 

    }
  • Mobile → 1 kolom
  • Tablet → 2 kolom
  • Desktop → 3 kolom

Tanpa duplikasi CSS.

Media Query selain width

Orientation

    @media (orientation: landscape) { 

        body { 

            background: red; 

        } 

    }

Height

    @media (min-height: 700px) { 

        .hero { 

            height: 80vh; 

        } 

    }

Media Query untuk input

Kita juga bisa cek:

    @media (hover: hover) { } 

    @media (pointer: coarse) { } /* layar sentuh */

Ini membuat UI adaptif:

  • hover effect di desktop
  • tap-friendly di mobile

Kesalahan umum

❌ Menulis semua CSS di dalam media query
❌ Memakai max-width terus
❌ Terlalu banyak breakpoint
❌ Meniru ukuran device, bukan konten

Aturan emas Responsive

Breakpoint bukan tentang device
Breakpoint tentang kapan layout terlihat jelek
