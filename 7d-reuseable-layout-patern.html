4. Reusable Layout Pattern Reusable Layout Pattern adalah pola tata letak yang bisa digunakan berulang di banyak halaman tanpa menulis ulang CSS dari nol. Ini bukan sekadar CSS, tapi cara berpikir sistematis tentang layout. Apa itu Layout Pattern? Layout pattern adalah struktur dasar halaman yang sering muncul, seperti: Header + Content Sidebar + Content Card Grid Dashboard Layout Form Layout Jika setiap halaman dibuat berbeda → sulit dirawat. Jika pakai pola → konsisten & scalable. Kenapa Reusable Layout itu penting? Tanpa reusable pattern: CSS duplikat desain tidak konsisten perubahan satu halaman merusak yang lain Dengan reusable pattern: satu perubahan → seluruh sistem ikut rapi lebih cepat develop lebih mudah dipelajari tim baru Prinsip Dasar Reusable Layout Pisahkan struktur & isi Gunakan wrapper/layout class Jangan campur komponen dengan layout Gunakan CSS variable untuk ukuran A. Layout Dasar: Page Wrapper .page { min-height: 100vh; display: flex; flex-direction: column; } HTML:
➡ Bisa dipakai di SEMUA halaman B. Layout Header + Content .header { height: var(--header-height); } .main { flex: 1; } Kelebihan: header tinggi konsisten konten otomatis mengisi sisa layar C. Sidebar Layout (Dashboard Pattern) .layout { display: flex; min-height: 100vh; } .sidebar { width: 260px; flex-shrink: 0; } .content { flex: 1; } HTML:
➡ Cocok untuk admin, dashboard, LMS D. Grid Content Layout .grid { display: grid; grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); gap: var(--space-md); } HTML:
➡ Responsive otomatis E. Container System (Max Width) .container { max-width: 1200px; padding-inline: var(--space-md); margin-inline: auto; } Tujuan: konten tidak terlalu lebar nyaman dibaca Best Practice Naming Fungsi Contoh Layout .layout, .page, .grid Area .header, .sidebar, .content Komponen .card, .button Kesalahan Umum ❌ Sidebar diberi style konten ❌ Card mengatur lebar halaman ❌ Layout bergantung pada isi ✔ Layout mengatur ruang ✔ Komponen mengisi ruang Hubungan dengan Design System Reusable layout: menggunakan CSS variables konsisten dengan design tokens mudah digabung dengan BEM & utility Ringkasan Layout ≠ Komponen Layout harus reusable Gunakan flex/grid Gunakan variable ukuran Pikirkan pola, bukan halaman
