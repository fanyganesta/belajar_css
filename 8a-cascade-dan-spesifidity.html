CSS Engineer (Mahir)

Apa itu CSS Engineer?

CSS Engineer adalah developer yang:

  • memahami cara browser memutuskan style
  • mampu mengontrol konflik CSS
  • menulis CSS scalable, performan, dan mudah di-debug
  • tahu anti-pattern dan menghindarinya
  • mampu memperbaiki layout rusak dengan cepat
CSS Engineer tidak menulis CSS berdasarkan trial & error, tapi berdasarkan aturan mesin CSS.

1. Cascade & Specificity (Advanced)

Ini adalah jantung CSS.
Jika kamu menguasai ini, 90% bug CSS bisa diprediksi sebelum terjadi.

Apa itu Cascade?

Cascade adalah aturan browser untuk menentukan:

CSS mana yang menang ketika beberapa aturan mengenai elemen yang sama.

Urutan keputusan browser:

  • Origin CSS (user agent, user, author)
  • Specificity
  • Order (urutan penulisan)
  • !important

1️⃣ CSS Origin (Asal CSS)

Urutan prioritas:

  • Browser default (User Agent)
  • User stylesheet
  • Author stylesheet (CSS kita)
  • Inline style
  • !important

    p { 

        color: red; 

    }

    <p style="color: blue">Teks</p>

✔ Yang dipakai: blue (inline lebih tinggi)

<h3>2️⃣ Specificity (Spesifisitas)</h3>

Specificity adalah nilai bobot selector.
Urutan Kekuatan Selector

Selector Bobot
Inline style 1000
ID (#id) 100
Class / attribute / pseudo-class 10
Element / pseudo-element 1

Contoh Perhitungan

    #header .menu a 

    #header → 100 

    .menu → 10

a → 1
Total: 111

Perbandingan Nyata

    a { color: red; }            /* 1 */ 

    .menu a { color: blue; }    /* 11 */ 

    #nav a { color: green; }    /* 101 */

✔ Yang menang: green

3️⃣ Order (Urutan Penulisan)

Jika specificity sama, maka:

aturan yang ditulis terakhir akan menang

    .button { 

        color: red; 

    } 
 


    .button { 

        color: blue; 

    }

✔ Hasil: blue

4️⃣ !important (Senjata Terakhir)

    .button { 

        color: red !important; 

    }

⚠ !important mengabaikan specificity

❌ Jangan digunakan sembarangan
✔ Hanya untuk:

  • override library pihak ketiga
  • utility class tertentu

Advanced Specificity Case

    .card h2 { color: red; }        /* 11 */ 

    .card .title { color: blue; }  /* 20 */

HTML:

    <h2 class="title">Judul</h2>

✔ Menang: .card .title

Specificity Trap (Jebakan Umum)

❌ Salah:

    #app .content .card .title span { 

        color: red; 

    }

Masalah:

  • specificity terlalu tinggi
  • sulit di-override
  • CSS jadi rapuh
✔ Benar:

    .card-title { 

        color: red; 

    }

Prinsip CSS Engineer tentang Specificity

✅ Gunakan class-based CSS
✅ Hindari ID untuk styling
✅ Specificity serendah mungkin
✅ Jangan menumpuk selector panjang
✅ Biarkan cascade bekerja, jangan dilawan

Strategi Aman Mengontrol Cascade

1. Layering CSS (Konseptual)

Urutan ideal:

  • Reset / Normalize
  • Base element
  • Layout
  • Component
  • Utility

2. Gunakan Utility untuk Override

    .text-center { 

        text-align: center; 

    }

Utility lebih aman daripada selector kompleks.

Checklist Engineer untuk Cascade

❓ Apakah selector ini perlu sepanjang ini?
❓ Apakah bisa pakai class?
❓ Apakah ini akan sulit dioverride?
❓ Apakah saya sedang melawan cascade?

Ringkasan Cascade & Specificity

  • Cascade = sistem keputusan browser
  • Specificity = bobot selector
  • Urutan penulisan tetap berpengaruh
  • !important adalah opsi terakhir
  • CSS Engineer mengontrol, bukan melawan
