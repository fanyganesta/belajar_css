Auto Placement & grid-auto-flow

Ini adalah sistem yang menentukan:

“Kalau aku tidak menentukan posisi item, CSS Grid harus meletakkannya di mana?”

Grid punya algoritma penempatan otomatis.

Default behavior

Jika kamu menulis:

    .container { 

        display: grid; 

        grid-template-columns: repeat(3, 1fr); 

    }

dan HTML:

    <div class="container"> 

    <div>A</div> 

    <div>B</div> 

    <div>C</div> 

    <div>D</div> 

    <div>E</div> 

    </div>

Grid akan mengisi seperti tabel:

    A | B | C 

    D | E | (kosong)

Ini disebut:
➡ row-based auto placement

grid-auto-flow

Properti ini menentukan arah pengisian.

    grid-auto-flow: row;     /* default */ 

    grid-auto-flow: column; 

    grid-auto-flow: dense;

1. grid-auto-flow: row (default)

Isi dari kiri → kanan, lalu turun ke baris berikutnya:

    A B C 

    D E F

Cocok untuk:

  • galeri
  • card
  • list produk

2. grid-auto-flow: column

Grid akan mengisi dari atas → bawah dulu:

    A D 

    B E 

    C F

Contoh:

    .container { 

        display: grid; 

        grid-template-rows: repeat(3, 100px); 

        grid-auto-flow: column; 

    }

Dipakai jika kamu ingin:

  • layout berbasis kolom
  • seperti koran atau timeline vertikal

3. grid-auto-flow: dense

Ini mengaktifkan “packing”

Jika ada item yang lompat:

    .item1 { grid-column: span 2; }
  • Maka Grid akan mencoba mengisi lubang kosong dengan item kecil berikutnya.
  • Bayangkan seperti Tetris otomatis.
    grid-auto-flow: row dense;

Ini bagus untuk:

  • masonry layout
  • galeri foto

Kenapa ini penting?

Karena kamu bisa:

  • membiarkan browser mengatur layout
  • tanpa menghitung posisi setiap item

Contoh:

    .card {
    grid-column: span 2;
    }

Grid akan:

“Oke, aku cari ruang kosong yang muat.”

Mental model

Flexbox: kamu menyuruh posisi
Grid auto-flow: kamu mendefinisikan aturan, browser yang menempatkan
