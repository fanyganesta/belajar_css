A. Width dan Constraint

1. Apa itu Width dan Constraint dalam CSS?

Dalam CSS, width menentukan seberapa lebar sebuah elemen, sedangkan constraint (pembatas) berfungsi untuk mengontrol batas minimum dan maksimum ukuran elemen agar layout tetap stabil di berbagai ukuran layar.

üìå Tanpa constraint, elemen bisa:

  • terlalu lebar di layar besar
  • rusak di layar kecil
  • menyebabkan overflow

2. Konsep Dasar Width

width mengatur lebar konten elemen (content box), bukan keseluruhan elemen. 

    .box {
        width: 300px; 

    }
Artinya:
  • konten lebarnya 300px
  • padding & border menambah ukuran total (jika box-sizing: content-box)

3. Mengapa Width Saja Tidak Cukup?

Masalah umum jika hanya menggunakan width:

a. Layar Terlalu Besar

    .container { 

        width: 1200px; 

    }

‚û° di layar besar tampak kecil
‚û° di layar kecil bisa overflow

b. Layout Tidak Responsif

Elemen tidak menyesuaikan dengan ukuran viewport.

üìå Inilah alasan constraint diperlukan.

4. Apa itu Constraint (Pembatas Ukuran)?

Constraint adalah aturan pembatas ukuran agar elemen:

  • tidak terlalu besar
  • tidak terlalu kecil
  • tetap proporsional

Constraint utama di CSS:

  • max-width
  • min-width
  • (nanti: height juga punya constraint)

5. Peran Constraint dalam Desain Modern

Desain modern jarang menggunakan width absolut saja.

Pola yang sering digunakan: 

    .container { 

        width: 100%; 

        max-width: 1200px; 

    }

Artinya:

  • lebar fleksibel
  • tapi tidak lebih dari 1200px

üìå Ini adalah best practice layout responsif

6. Hubungan Width dan Parent Element

Jika width menggunakan satuan relatif:

    .box { 

        width: 50%; 

    }

Maka:

  • lebar dihitung dari parent element
  • bukan dari viewport (kecuali parent = viewport)

üìå Kesalahan umum: mengira % selalu dari layar

7. Width, Content, dan Flow Layout

Lebar elemen memengaruhi:

  • pembungkusan teks (text wrapping)
  • tinggi otomatis elemen
  • posisi elemen lain
    .card { 

        width: 300px; 

    }

‚û° teks panjang akan turun baris
‚û° tinggi elemen bertambah otomatis

8. Constraint Mencegah Layout Rusak

Tanpa constraint:

  • gambar bisa terlalu besar
  • teks terlalu panjang
  • container meluber

Dengan constraint:

    img {  

        max-width: 100%; 

    }

üìå Ini mencegah gambar keluar dari container

9. Constraint dan Responsiveness

Constraint adalah fondasi responsive design.

Contoh pola umum:

    .section { 

        width: 100%; 

        max-width: 1100px; 

        margin: 0 auto; 

    }

Hasil:

  • mobile: full width
  • desktop: lebar dibatasi
  • layout tetap rapi

10. Perbedaan Konsep: Width vs Constraint

Aspek Width Constraint
Fungsi Menentukan ukuran Membatasi ukuran
Sifat Tetap / fleksibel Protektif
Wajib Tidak Sangat disarankan
Efek ke layout Langsung Preventif

11. Kesalahan Umum Pemula

‚ùå Mengunci width besar tanpa max-width
‚ùå Menggunakan px untuk semua ukuran
‚ùå Mengabaikan layar kecil
‚ùå Tidak memahami parent width

12. Pola Best Practice Dasar

‚úî Gunakan width + max-width
‚úî Gunakan % atau vw dengan pembatas
‚úî Biarkan height otomatis
‚úî Gunakan constraint untuk stabilitas

B. Width dan Max-Width (Pembahasan Mendalam)

1. Apa itu width?

width menentukan lebar dasar sebuah elemen. 

    .box { 

        width: 400px; 

    }

Artinya:

  • konten elemen lebarnya 400px
  • tidak peduli ukuran layar
  • elemen dipaksa selebar itu
üìå width bersifat mengunci ukuran, kecuali memakai satuan fleksibel.

2. Bagaimana CSS Menghitung Width?

Secara default (box-sizing: content-box):

total lebar = width + padding + border

Contoh: 

    .box { 

        width: 300px; 

        padding: 20px; 

        border: 5px solid black; 

    }

‚û° total lebar = 300 + 40 + 10 = 350px

üìå Ini sering menyebabkan overflow tanpa disadari.

3. Jenis Nilai width

a. Nilai Absolut

    width: 300px;

‚úî presisi
‚ùå tidak responsif

b. Nilai Relatif ke Parent

width: 50%;

‚úî fleksibel
‚ùå bisa terlalu besar di layar lebar

c. Nilai Relatif ke Viewport

    width: 80vw;

‚úî adaptif
‚ùå berisiko terlalu lebar

4. Masalah Umum Jika Hanya Menggunakan width

‚ùå Elemen Terlalu Lebar di Layar Besar 

    .container { 

        width: 100%; 

    }

‚û° di layar 4K, konten terlalu melebar
‚û° sulit dibaca

‚ùå Overflow di Layar Kecil 

    .container { 

        width: 1200px; 

    }

‚û° mobile akan scroll horizontal

5. Apa itu max-width?

max-width adalah batas maksimum lebar elemen. 

    .container { 

        max-width: 1200px; 

    }

Artinya:

  • elemen boleh mengecil
  • tetapi tidak boleh lebih lebar dari 1200px
üìå max-width tidak mengunci, hanya membatasi.

6. Perbedaan Perilaku width vs max-width

Kondisi Layar width: 1200px max-width: 1200px
Layar kecil Overflow Menyusut
Layar besar Tetap 1200px Tetap 1200px
Fleksibel ‚ùå ‚úî

7. Kombinasi Terbaik: width + max-width

Pola paling umum dan direkomendasikan:

    .container { 

        width: 100%; 

        max-width: 1200px; 

    }

Artinya:

  • mobile ‚Üí full width
  • desktop ‚Üí dibatasi
  • layar besar ‚Üí tetap nyaman dibaca
üìå Ini adalah fondasi layout responsif

8. Mengapa max-width Lebih Aman dari width?

Karena:

  • tidak memaksa ukuran
  • mengikuti ukuran layar
  • mencegah overflow
  • cocok untuk semua device

    img { 

        max-width: 100%; 

    }

‚û° gambar tidak pernah keluar container

9. Urutan Prioritas CSS

Jika width dan max-width digunakan bersama:

    .box { 

        width: 1000px; 

        max-width: 600px; 

    }

‚û° yang berlaku: max-width (600px)

üìå Constraint selalu menang jika dilanggar.

10. max-width dengan Satuan Relatif

    .container { 

        max-width: 90%; 

    }

‚û° tetap fleksibel
‚û° tapi jarang dipakai untuk layout utama

Lebih umum:

max-width: 1100px;

11. Contoh Kasus Nyata

Layout Artikel 

    .article { 

        width: 100%; 

        max-width: 700px; 

        margin: 0 auto; 

    }

‚û° teks nyaman dibaca
‚û° tidak terlalu panjang
‚û° mobile friendly

12. Kesalahan Umum

‚ùå Menggunakan width besar tanpa max-width
‚ùå Mengunci layout dengan px semua
‚ùå Tidak mengatur gambar dengan max-width
‚ùå Menganggap % selalu aman

13. Best Practice Ringkas

‚úî Gunakan width: 100% untuk fleksibilitas
‚úî Tambahkan max-width sebagai pengaman
‚úî Gunakan margin: auto untuk center
‚úî Hindari width absolut untuk layout utama

C. Height dan Problemnya

1. Apa itu height?

height menentukan tinggi sebuah elemen.

    .box { 

        height: 200px; 

    }

Artinya:

  • tinggi elemen dipaksa 200px
  • konten di dalamnya tidak mempengaruhi tinggi
üìå Berbeda dengan width, tinggi alami elemen biasanya ditentukan oleh kontennya.

2. Perbedaan Dasar width vs height

Properti Default behavior
width mengikuti parent
height mengikuti konten

‚û° Karena itu, mengatur height jauh lebih berisiko

3. Masalah Utama Saat Menggunakan height

‚ùå Konten Terpotong

    .card { 

        height: 150px; 

    }

Jika konten lebih tinggi:

  • teks terpotong
  • elemen lain rusak
  • UX buruk
‚ùå Layout Tidak Responsif 

    .section { 

        height: 600px; 

    }

layar kecil ‚Üí terlalu tinggi

layar besar ‚Üí terlihat kosong

üìå height absolut tidak adaptif

4. Masalah Paling Umum: height: 100%

    .box { 

        height: 100%; 

    }

‚û° Sering tidak bekerja

Kenapa?

Karena:

  • height: 100% bergantung pada height parent
Jika parent tidak memiliki height eksplisit, maka:
  • browser tidak tahu 100% itu dari apa
  • hasilnya = auto
Contoh Gagal 

    .parent { 

        /* tidak ada height */ 

    } 

        

    .child { 

        height: 100%; 

    }

‚û° child tidak akan penuh

Contoh Benar 

    html, body { 

        height: 100%; 

    } 

    

    .parent { 

        height: 100%; 

    } 

        

    .child { 

        height: 100%; 

    }

üìå Semua level harus punya height.

5. height vs min-height (Solusi Penting)

‚ùå height (berisiko)

    .box { 

        height: 300px; 

    }

‚úî min-height (aman)

    .box { 

        min-height: 300px; 

    }

Artinya:

  • minimal 300px
  • boleh lebih tinggi jika konten bertambah
üìå min-height hampir selalu lebih aman

6. Studi Kasus: Section Full Screen

‚ùå Salah

                .hero { 

                    height: 100%; 

                }

‚úî Benar

    .hero { 

        min-height: 100vh; 

    }

‚û° selalu setinggi layar
‚û° konten tetap aman

7. vh dan Masalahnya

    .section { 

        height: 100vh; 

    }

Masalah:

  • mobile browser punya address bar dinamis
  • tinggi viewport berubah
  • konten bisa terpotong
üìå Terutama di Android & iOS lama.

Solusi Aman 

    .section { 

        min-height: 100vh; 

    }

8. Height dalam Flexbox

Flexbox mengubah cara height bekerja.

    .container { 

        display: flex; 

        height: 100vh; 

    } 

        

    .sidebar { 

        width: 250px; 

    } 

        

    .content { 

        flex: 1; 

    }

‚û° flex item bisa otomatis stretch vertikal
‚û° tanpa set height manual

üìå Flexbox = solusi modern untuk masalah height

9. Height dan Scroll

Jika height dibatasi:

    .box { 

        height: 300px; 

        overflow: auto; 

    }

‚û° konten panjang akan scroll
‚û° bukan memecah layout

10. Kesalahan Umum (WAJIB DIHINDARI)

‚ùå Mengunci height konten dinamis
‚ùå Menggunakan height: 100% tanpa parent height
‚ùå Menggunakan px untuk layout vertikal
‚ùå Mengabaikan min-height

11. Best Practice Height

‚úî Biarkan tinggi mengikuti konten
‚úî Gunakan min-height, bukan height
‚úî Gunakan vh untuk section besar
‚úî Manfaatkan Flexbox/Grid
‚úî Gunakan overflow jika perlu
